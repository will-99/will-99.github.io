基本概念

Generator函数是ES6提供的一种异步编程解决方案。Generator函数只有调用next方法时，函数才会真正的执行。

Generator函数有两个特征：

  1. function关键字与函数名之间有一个星号。
  2. 函数体内部用yield表达式，定义不同的内部状态。

Generator()函数这样被调用后，该函数并执行，返回的是一个指向内部状态的指针对象。下一步必须调用遍历器对象的next方法，使得指针移向下一个状态。每次调用next方法，内部指针就从函数头部或上一次停下来的地方开始执行，直到遇到yield或return。Generator函数是分段执行的，遇到yield会暂停执行，next方法可以恢复执行。

yield表达式

遍历器对象的next方法的运行逻辑如下：

  1. 遇到yield表达式就暂停执行，并将yield后面的表达式，作为返回对象的value的属性值。
  2. 下一次调用next方法则继续往下执行，直到遇到下一个yield。
  3. 如果没有再遇到新的yield表达式，就一直运行到函数结束，直到return语句为止，并将return语句后面的表达式的值，作为返回的对象的value属性值。
  4. 如果没有return语句，则返回的对象的value属性值为undefined。

next方法的参数

yield表达式本身没有返回值，即返回的总是undefined。next方法可以带一个参数，该参数就会被当做上一个yield表达式的返回值。
